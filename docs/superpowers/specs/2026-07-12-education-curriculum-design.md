# KPOPSOFT 교육 트랙형 커리큘럼 설계

작성일: 2026-07-12  
대상 브랜치: `codex/kpopsoft-maxonomy-concept-wind`

## 1. 목표

현재 교육 섹션의 8개 프로그램 단순 나열을, 방문 목적에 따라 빠르게 찾을 수 있는 6개 교육 트랙으로 재구성한다. 각 트랙은 `입문 → 실무 → 프로젝트`의 동일한 학습 흐름을 사용한다. 데스크톱과 모바일에서 같은 정보 구조를 유지하되, 모바일에서는 한 번에 한 트랙만 펼치는 아코디언으로 화면 길이와 탐색 부담을 줄인다.

이번 범위는 공개 홈페이지의 교육 콘텐츠와 교육 문의 연결이다. 관리자용 교육 CMS, 결제, 수강 신청, 학습 진도 관리, 회원 기능은 포함하지 않는다.

## 2. 정보 구조

### 트랙 01 — AI 기초교육

설명: AI를 처음 접하는 사람이 생성형 AI의 기본 원리와 안전한 사용법을 익히는 과정.

- 입문: 생성형 AI 이해와 활용 사례
- 실무: 프롬프트 기초, ChatGPT 및 주요 AI 도구 사용
- 프로젝트: 개인 업무에 적용할 AI 활용 가이드 만들기
- 공통 필수: 개인정보, 보안, 저작권, 결과 검증
- 수료 결과: AI 도구를 스스로 선택하고 기본 업무에 안전하게 적용할 수 있다.

### 트랙 02 — 업무용 AI

설명: 문서·회의·리서치·데이터 정리 등 반복되는 사무 업무의 생산성을 높이는 과정.

- 입문: 직무별 AI 활용 사례와 업무 분해
- 실무: 문서 작성·요약·번역, 회의록, 보고서, 리서치, 데이터 정리
- 프로젝트: 자신의 직무에 맞는 재사용 프롬프트 템플릿 제작
- 수료 결과: 반복적인 지식 업무에 사용할 표준 AI 작업 절차를 만들 수 있다.

### 트랙 03 — 바이브 코딩

설명: 비개발자와 기획자가 AI 코딩 도구로 실제 작동하는 웹페이지와 업무 도구를 만드는 과정.

- 입문: AI 코딩 도구 설정, 요구사항 작성, 작업 단위 나누기
- 실무: 웹페이지·폼·간단한 업무 도구 제작, 오류 수정
- 프로젝트: 공개 가능한 프로토타입 제작과 배포
- 수료 결과: 아이디어를 직접 테스트 가능한 디지털 프로토타입으로 구현할 수 있다.

### 트랙 04 — AI 자동화

설명: 반복 업무를 찾아 노코드 도구와 AI를 연결한 자동화 흐름으로 바꾸는 과정.

- 입문: 자동화 후보 업무 발굴과 현재 흐름 진단
- 실무: 노코드 자동화, 문서·메일·데이터 연결, 예외 처리
- 프로젝트: 실제 업무 자동화 워크플로 1개 구축
- 수료 결과: 팀의 반복 업무 하나를 안정적으로 실행되는 자동화 흐름으로 전환할 수 있다.

### 트랙 05 — 콘텐츠·동영상 제작

설명: AI를 활용해 콘텐츠 기획부터 이미지·음성·자막·영상 편집까지 이어가는 제작 과정.

- 입문: 콘텐츠 목적 설정, 아이디어 발굴, 대본과 스토리보드 작성
- 실무: 이미지 제작, AI 음성, 자막, 장면 구성, 편집과 품질 검수
- 프로젝트: 숏폼 또는 교육용 동영상 1편 완성
- 수료 결과: 반복 가능한 AI 콘텐츠 제작 흐름과 게시 가능한 영상 결과물을 만들 수 있다.

### 트랙 06 — 기업 맞춤·프로젝트

설명: 조직의 직무와 실제 현업 과제에 맞춰 커리큘럼과 결과물을 함께 설계하는 기업 전용 과정.

- 입문: 조직·직무별 역량과 현업 과제 진단
- 실무: 부서별 맞춤 워크숍과 실제 업무 데이터 실습
- 프로젝트: 사내 과제 결과물 제작, 성과 점검, 확산 가이드
- 수료 결과: 교육 내용을 실제 업무 개선 과제와 팀 표준으로 연결할 수 있다.

## 3. 화면 설계

### 공통

- 섹션 제목과 소개 문구는 현재 KPOPSOFT 타이포그래피와 여백 체계를 유지한다.
- 트랙은 `01`부터 `06`까지 고정 순서로 보여준다.
- 각 트랙 헤더에는 번호, 트랙명, 한 문장 설명, 핵심 태그를 배치한다.
- 트랙을 펼치면 `입문`, `실무`, `프로젝트` 3단계와 수료 결과를 보여준다.
- 첫 번째 트랙은 최초 진입 시 펼쳐져 있고, 나머지는 닫혀 있다.
- 한 트랙을 열면 이전에 열려 있던 트랙은 닫힌다.
- 트랙 하단 CTA는 `이 교육 상담하기`이며 공개 문의 폼의 `교육 문의`로 이동한다.
- 색상만으로 트랙이나 단계를 구분하지 않는다. 모든 상태는 텍스트와 `aria-expanded`를 함께 사용한다.

### 데스크톱·태블릿

- 트랙 헤더는 기존 교육 섹션의 전체 폭 행과 헤어라인 구분을 재사용한다.
- 펼친 영역은 3열로 배치해 `입문 → 실무 → 프로젝트` 흐름을 왼쪽에서 오른쪽으로 읽게 한다.
- 수료 결과와 상담 CTA는 3열 아래의 한 줄 영역에 둔다.
- 새로운 카드 그리드나 이미지 자산은 추가하지 않는다.

### 모바일

- 기준 폭은 390px이며 좌우 콘텐츠 여백은 최소 20px을 유지한다.
- 트랙 헤더는 번호, 제목, 펼침 아이콘을 첫 줄에 두고 설명은 다음 줄에 둔다.
- 펼친 영역은 `입문 → 실무 → 프로젝트 → 수료 결과 → CTA` 순서로 한 열에 쌓는다.
- 트랙 헤더와 CTA의 터치 높이는 최소 48px이다.
- 긴 한글 제목이 잘리거나 가로 스크롤을 만들지 않는다.
- 펼침 아이콘은 기존 Lucide 아이콘을 사용하고, 문자 기호나 새 장식 이미지는 만들지 않는다.

## 4. 데이터 구조

`src/lib/site.ts`의 `programs` 단순 배열을 `educationTracks`로 교체한다. 한 트랙은 다음 계약을 가진다.

```ts
type EducationStage = {
  level: "입문" | "실무" | "프로젝트";
  title: string;
  modules: readonly string[];
};

type EducationTrack = {
  id:
    | "ai-foundations"
    | "work-ai"
    | "vibe-coding"
    | "ai-automation"
    | "content-video"
    | "custom-project";
  index: string;
  title: string;
  description: string;
  tags: readonly string[];
  accent: Accent;
  stages: readonly [EducationStage, EducationStage, EducationStage];
  outcome: string;
  inquirySubtype: string;
};
```

`src/components/sections/education.tsx`는 `educationTracks`만 읽는다. 별도 데이터 복사본을 컴포넌트 안에 만들지 않는다.

## 5. 문의 연결

교육 트랙 CTA는 다음 스냅샷 label을 사용한다.

| 트랙 | 문의 유형 | 세부 유형 |
|---|---|---|
| AI 기초교육 | 교육 문의 | AI 기초교육 |
| 업무용 AI | 교육 문의 | 업무용 AI |
| 바이브 코딩 | 교육 문의 | 바이브 코딩 |
| AI 자동화 | 교육 문의 | AI 자동화 |
| 콘텐츠·동영상 제작 | 교육 문의 | 콘텐츠·동영상 제작 |
| 기업 맞춤·프로젝트 | 교육 문의 | 기업 맞춤·프로젝트 |

`src/lib/site.ts`의 정적 `inquiryOptions` 교육 세부 유형도 위 6개와 `기타`로 맞춘다. 기존 접수 건은 문자열 스냅샷이므로 과거 label을 그대로 유지한다. 운영 Supabase의 `inquiry_subtypes`가 정적 폴백보다 우선하므로, 배포 전 관리자 화면에서 동일한 6개 label과 placeholder로 동기화해야 한다. 원격 DB 변경은 이 작업에서 자동 적용하지 않는다.

`src/components/sections/b2b-education.tsx`의 CTA는 `기업 맞춤·프로젝트`를 선택하도록 변경한다.

## 6. 상태와 오류 처리

- 아코디언 상태는 클라이언트에서 `openTrackId` 하나로 관리한다.
- 존재하지 않는 트랙 ID는 열지 않고 현재 상태를 유지한다.
- JavaScript가 로드되기 전에도 트랙 헤더와 설명은 읽을 수 있어야 한다.
- 문의 URL의 세부 유형이 운영 DB 옵션과 일치하지 않으면 문의 유형만 선택하고 세부 유형은 사용자가 다시 고르게 한다. 폼 제출 자체를 막지 않는다.
- 커리큘럼 데이터는 빌드 타임 계약 테스트에서 누락과 중복을 차단한다.

## 7. 접근성

- 트랙 헤더는 실제 `button` 요소를 사용한다.
- `aria-expanded`, `aria-controls`, 패널 `id`를 연결한다.
- 키보드 Tab과 Enter/Space로 열고 닫을 수 있다.
- 패널을 열 때 포커스를 강제로 이동하지 않는다.
- 애니메이션은 높이와 투명도를 짧게 사용하고 `prefers-reduced-motion`에서는 제거한다.

## 8. 검증 기준

- 데이터 계약 테스트에서 트랙이 정확히 6개이고 ID·index·title이 중복되지 않아야 한다.
- 모든 트랙은 `입문`, `실무`, `프로젝트`를 정확한 순서로 한 번씩 가져야 한다.
- 각 단계는 최소 1개의 모듈을 가져야 한다.
- 모든 `inquirySubtype`은 정적 `교육 문의` 옵션에 존재해야 한다.
- 데스크톱 1440px, 태블릿 768px, 모바일 390px에서 가로 스크롤과 잘림이 없어야 한다.
- 첫 트랙 기본 펼침, 단일 펼침, 재클릭 닫기, CTA 문의 이동을 브라우저에서 확인한다.
- `npm test`, `npm run lint`, `npm run build`가 모두 통과해야 한다.

## 9. 범위 밖

- 교육용 이미지 생성
- 교육 프로그램 관리자 CRUD 및 별도 DB 테이블
- 수강 신청·결제·회원·진도 관리
- 운영 Supabase 데이터의 자동 마이그레이션
- 강의별 상세 라우트
