# KPOPSOFT 사진 자산 분리·배치 설계

## 목표

첨부된 10개 장면을 하나의 콜라주로 사용하지 않고 각각 독립된 고화질 사진 자산으로 분리한다. 기존 KPOPSOFT 로고, 아이보리 기반 편집형 화면, 브랜드 색상과 교육 아코디언의 구조는 유지하면서 회사·개발·교육 맥락에 맞춰 사진을 분산 배치한다. 기존 강사 프로필 사진도 Experts 영역의 핵심 시각 자산으로 적극 활용한다.

## 승인된 방향

선택안은 A형 `맥락별 분산 배치`다. 사진만 모은 별도 갤러리를 만들거나 사진을 전체 섹션 배경으로 사용하지 않는다. 각 장면이 설명하는 업무와 가장 가까운 기존 섹션 안에서 사진을 보여준다.

## 자산 분리와 품질 기준

원본 콜라주의 장면 10개를 다음 독립 자산으로 만든다.

1. 개발 협업 회의
2. 노트북 대시보드 작업
3. 소규모 실무 교육
4. KPOPSOFT 로고 벽면
5. 라운지 팀 협업
6. 듀얼 모니터 개발 환경
7. 강의장 교육
8. KPOPSOFT 사옥 전경
9. 제품 설계 스케치
10. 유리 회의실 팀 협업

각 자산은 원본 장면의 인물, 구도, 화면, 공간, 로고를 바꾸지 않는 것을 최우선으로 한다. 개별 패널 해상도가 웹 표시 목표보다 낮으면 고화질 보정을 적용하되 새로운 인물·사물·문구를 만들지 않는다. 특히 KPOPSOFT 로고가 포함된 장면은 글자와 심볼의 형태를 그대로 보존한다.

프로젝트 최종 자산은 `public/images/kpopsoft/` 아래에 의미가 드러나는 영문 파일명으로 저장한다. 각 이미지는 실제 표시 슬롯에 맞는 가로·세로 비율로 준비하고 불필요한 재크롭을 피한다.

## 섹션별 배치

### Company Introduction

- KPOPSOFT 로고 벽면을 주 이미지로 사용한다.
- 사옥 전경을 보조 이미지로 사용한다.
- 소개 문구 아래에 7:5 비율의 편집형 2열 레이아웃을 둔다.
- 모바일에서는 로고 벽면, 사옥 전경 순서로 세로 배치한다.
- 헤더의 기존 로고는 변경하지 않는다.

### Software

- 개발 협업 회의, 노트북 대시보드, 듀얼 모니터 개발 환경, 제품 설계 스케치를 사용한다.
- 기존 서비스 문의 카드와 CTA는 유지한다.
- 서비스 카드 아래에 `현장에서 만드는 과정` 성격의 편집형 이미지 스트립을 추가한다.
- 개발 협업 사진을 가장 크게 두고 나머지 3개 장면을 보조 리듬으로 배치한다.
- 이미지 자체를 링크나 CTA로 오인하지 않도록 사진 영역은 정보성 콘텐츠로 유지한다.

### Education

- 소규모 실무 교육과 강의장 교육 장면을 사용한다.
- 기존 교육 헤딩과 6개 트랙 아코디언 사이에 2장으로 구성된 교육 현장 프리뷰를 둔다.
- 실무 교육은 넓은 주 이미지, 강의장 교육은 보조 이미지로 사용한다.
- 아코디언과 과정별 상담 CTA의 동작은 변경하지 않는다.

### Experts

- 기존 강사 프로필 사진을 계속 사용하며 현재보다 더 큰 초상 이미지 비중을 준다.
- 인물 사진의 원본 비율과 얼굴 중심 구도를 보존한다.
- 강사별 이름, 역할, 인용문, 태그 구조는 유지한다.
- 프로필 사진이 없는 강사에 대해서만 기존 모노그램 대체 UI를 사용한다.
- 교육 현장 사진은 Education의 맥락을, 강사 사진은 개별 전문성을 전달하도록 역할을 분리한다.

### B2B Education

- 라운지 팀 협업과 유리 회의실 팀 협업 장면을 사용한다.
- 기존 네이비 CTA 패널의 왼쪽에는 문구와 상담 버튼을 유지하고 오른쪽에 협업 사진 2장을 배치한다.
- 텍스트 위에 사진을 겹치거나 사진을 어둡게 덮어 배경으로 사용하지 않는다.
- 모바일에서는 텍스트와 CTA 다음에 사진 2장을 세로 배치한다.

## 반응형·접근성

- `next/image`를 사용해 `sizes`와 지연 로딩을 적용한다.
- 모든 사진에 장면을 설명하는 한국어 대체 텍스트를 제공한다.
- 모바일 390px, 태블릿 768px, 데스크톱 1440px에서 가로 넘침이 없어야 한다.
- 인물의 얼굴, 로고, 프레젠테이션 화면이 임의로 잘리지 않도록 슬롯별 `object-position`을 조정한다.
- 사진 모서리와 간격은 현재 사이트의 `rounded-3xl`, 헤어라인, 아이보리 여백 체계를 따른다.
- 사진 추가로 기존 버튼의 최소 높이, 키보드 포커스, 상담 딥링크가 손상되지 않아야 한다.

## 구현 경계

- 새 페이지나 별도 사진 갤러리 라우트를 만들지 않는다.
- 헤더 로고, 브랜드 색상, 서체, 교육 트랙 데이터와 문의 처리 로직은 변경하지 않는다.
- 이미지 자산과 이를 소비하는 섹션 컴포넌트만 필요한 범위에서 수정한다.
- 현재 브랜치 `codex/kpopsoft-maxonomy-concept-wind`와 작업공간을 유지하며 병합·푸시하지 않는다.

## 검증

- 10개 분리 자산과 강사 이미지 참조가 모두 존재하는지 테스트한다.
- `npm test`, `npm run lint`, `npm run build`, `git diff --check`를 실행한다.
- Chrome에서 390px, 768px, 1440px 화면을 확인한다.
- 회사 소개, Software, Education, Experts, B2B Education의 사진 배치와 가독성을 시각 검토한다.
- 교육 아코디언 전환과 과정별 상담 딥링크를 다시 확인한다.
