# KPOPSOFT 사진 9장 교체 설계

## 목표

2026년 7월 13일에 전달받은 새 사진 9장으로 홈페이지의 편집형 현장 사진을 전부 교체한다. 사진은 중복 사용하지 않고 각 장면을 정확히 한 번씩 배치한다. 기존 KPOPSOFT 헤더 로고, 브랜드 색상, 강사 프로필 사진, 교육 아코디언과 문의 CTA 동작은 변경하지 않는다.

## 승인된 구성

선택안은 A안이다.

- 회사 소개: 1장
- 소프트웨어: 4장
- 교육: 2장
- B2B 교육: 2장
- 합계: 9장

기존 회사 소개의 2장 사진 행은 1장의 넓은 대표 사진으로 바꾼다. 나머지 섹션의 사진 수와 정보 흐름은 유지한다.

## 자산 처리

- 첨부 PNG의 인물, 화면, 구도와 공간을 변경하지 않는다.
- AI 재생성이나 생성형 업스케일을 사용하지 않는다.
- 원본의 긴 변이 1600px보다 크면 1600px로 축소하고, 더 작으면 확대하지 않는다.
- 웹 자산은 JPEG 품질 92 이상으로 변환해 `public/images/kpopsoft/`에 저장한다.
- 삭제되는 `about-headquarters.jpg`와 이전 현장 사진은 새 화면에서 참조하지 않는다.
- `src/lib/photography.ts`의 경로, 장면 설명형 한국어 대체 텍스트와 필요한 초점 위치를 새 사진에 맞춘다.

## 레이아웃

### Company Introduction

오픈 오피스 사진 한 장을 소개 문구 아래의 넓은 대표 이미지로 사용한다. 기존 7:5 두 열 사진 행은 단일 열로 바꾸며, 데스크톱과 모바일 모두 사진 속 앞쪽 회의 그룹과 뒤쪽 협업 공간이 함께 보이도록 한다.

### Software

협업 발표 사진을 가장 크게 배치한다. AI 대시보드, 듀얼 모니터 개발 환경, 화면 설계 스케치는 기존 3개 보조 슬롯에 배치한다. 기존 서비스 카드와 문의 CTA는 그대로 유지한다.

### Education

AI 워크플로 강의실을 넓은 주 사진으로, 소규모 멘토링을 보조 사진으로 배치한다. 사진 아래의 6개 교육 트랙 아코디언과 과정별 문의 딥링크는 유지한다.

### Experts

기존 안영근·김상혁 강사 사진과 프로필 카드, 이미지 미등록 강사의 모노그램 대체 UI를 그대로 유지한다. 새 현장 사진으로 강사 프로필을 교체하지 않는다.

### B2B Education

회의실 화이트보드 워크숍을 주 사진으로, 라운지 대화를 보조 사진으로 사용한다. 기존 네이비 패널의 문구, 상담 버튼과 모바일 DOM 순서는 유지한다.

## 반응형과 접근성

- `EditorialPhoto`와 `next/image`의 현재 로딩·`sizes` 계약을 유지한다.
- 390px, 768px, 1440px에서 가로 넘침이 없어야 한다.
- 사진의 얼굴, 노트북, 발표 화면이 슬롯 경계에서 부자연스럽게 잘리지 않도록 `object-position`을 조정한다.
- 모든 대체 텍스트는 사진의 실제 장면을 설명하며 회사명이나 인물 신원을 추정하지 않는다.
- 사진은 정보성 콘텐츠로 유지하고 링크나 버튼으로 만들지 않는다.

## 구현 경계

- 새 페이지, 갤러리, 데이터 모델 또는 관리 기능을 추가하지 않는다.
- 헤더 로고, 브랜드 색상, 서체, 강사 데이터, 교육 트랙과 문의 처리 로직을 변경하지 않는다.
- 이미지 자산, 사진 매니페스트, 회사 소개의 사진 그리드와 관련 계약 테스트·문서만 필요한 범위에서 수정한다.
- 현재 브랜치 `codex/kpopsoft-maxonomy-concept-wind`를 유지하며 병합하거나 원격으로 푸시하지 않는다.

## 검증

- 새 사진 9개와 기존 강사 이미지 2개의 경로가 모두 유효한지 확인한다.
- 사진 매니페스트가 정확히 9개의 중복 없는 자산을 내보내는지 테스트한다.
- 기존 `EditorialPhoto` 동작과 각 섹션의 사진 연결 계약을 확인한다.
- `npm test`, `npm run lint`, `npm run build`와 브랜치 범위 `git diff --check`를 실행한다.
- Chrome에서 390px, 768px, 1440px 화면과 교육 아코디언·문의 딥링크를 다시 검증한다.
