## 운영 배포 HOLD

다음 항목은 로컬 코드만으로 완료했다고 판단하지 않습니다. 현재 의존성과 설정에는 여러 인스턴스가 공유하는 durable rate/quota 저장소나 challenge 공급자가 없으므로, 프로세스 메모리 기반 제한은 추가하지 않았습니다.

1. **문의 남용 방지(HOLD)**
   - 운영 도메인에 challenge 공급자(예: Cloudflare Turnstile)를 프로비저닝하고 site key/secret을 배포 환경에 등록합니다.
   - 문의 UI가 받은 challenge token을 `submitInquiry()`에서 공급자 서버 API로 검증하고, 검증 성공 전에는 DB insert와 이메일 전송을 모두 금지합니다.
   - Supabase 원자적 RPC/테이블 또는 관리형 rate-limit 저장소 중 하나를 프로비저닝해 모든 앱 인스턴스가 공유하도록 합니다. 운영자가 시간창·허용량·차단 응답을 확정한 뒤, insert/email 전에 원자적으로 consume하며 동시 요청과 다중 인스턴스 테스트를 통과해야 합니다.
2. **문의 RLS(HOLD)**
   - 인증된 Supabase CLI/MCP로 새 마이그레이션을 생성해 `inquiries_insert_anon` 정책을 제거하고 `anon, authenticated`의 직접 INSERT 권한을 revoke합니다. 공개 문의 저장은 검증된 Server Action의 service-role 경로로만 허용합니다.
   - `public.is_admin()`의 `PUBLIC` execute를 revoke하고 필요한 역할에만 grant한 뒤 Supabase advisors와 역할 매트릭스(anon INSERT/SELECT 거부, 비관리자 쓰기 거부, 관리자 조회/수정 허용, service-role INSERT 허용)를 실행합니다.
3. **운영 콘텐츠 동기화(HOLD)**
   - 공개 페이지는 비어 있지 않은 운영 Supabase 행을 정적 seed보다 우선합니다. 배포 전 `experts`의 이름·역할·인용·태그·이미지·정렬·게시 상태를 `src/lib/site.ts`의 승인본과 대조합니다.
   - `교육 문의`의 활성 `inquiry_subtypes`를 6개 트랙(`AI 기초교육`, `업무용 AI`, `바이브 코딩`, `AI 자동화`, `콘텐츠·동영상 제작`, `기업 맞춤·프로젝트`)과 `기타`의 label/placeholder/정렬로 동기화하고, 모든 교육 CTA가 예상 subtype을 선택하는지 운영 도메인에서 확인합니다.
