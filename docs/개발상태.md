# 개발 상태

마지막 확인: 2026-07-12

## 현재 상태

- Next.js 16 App Router 기반 KPOPSOFT 랜딩페이지가 구현되어 있습니다.
- 기획서의 16개 구성(Header, Hero, 주요 섹션, Final CTA, Footer)이 `src/app/page.tsx`에 순서대로 조립되어 있습니다.
- 공개 페이지 콘텐츠는 아직 DB가 아니라 `src/lib/site.ts`의 정적 데이터입니다.
- Final CTA 문의 폼은 mailto 대신 Server Action(`src/lib/inquiry-actions.ts`)으로 접수 + 이메일 알림(stub)합니다.
- Education은 AI 기초교육, 업무용 AI, 바이브 코딩, AI 자동화, 콘텐츠·동영상 제작, 기업 맞춤·프로젝트의 6개 트랙으로 구성되어 있습니다. 각 트랙은 반응형 아코디언 안에서 `입문 → 실무 → 프로젝트` 커리큘럼과 과정별 상담 CTA를 제공합니다.

- **P2 완료** — `/admin/content/{work,insights,testimonials,experts,stats,inquiry-options}` 등록/수정/삭제/노출 토글. 강사·Work·Insights 커버 이미지 업로드(mock: data URL, 미리보기 동작). 문의 옵션은 유형→세부 유형(예시 문구) 2단 관리.

- Supabase Auth/DB/Storage/이메일 발송은 아직 미연결(나중에 일괄).

## 검증

- `npm test` 통과 (17개)
- `npm run lint` 통과
- `npm run build` 통과

Next.js 16 기본 Turbopack 빌드는 이 Codex 샌드박스에서 내부 포트 바인딩 제한 때문에 실패했습니다. `node_modules/next/dist/docs/01-app/03-api-reference/08-turbopack.md` 기준으로 Webpack 빌드가 공식 대안이라 `npm run build`는 `next build --webpack`을 사용하도록 맞춰두었습니다.
