Create a complete responsive website design system for KPOPSOFT.

KPOPSOFT is a software development and AI education company.

The company provides:

- AI 업무 활용 교육
- Vibe Coding
- Software Development
- Web & App Development
- AI Automation
- AI Prototype Lab
- Digital Product Development
- Corporate Custom Training

KPOPSOFT also operates with a professional instructor and expert network covering:

- AI 활용
- Vibe Coding
- Software Development
- AI Automation
- Digital Product
- Prototype Development


## DESIGN DIRECTION

Create a playful, editorial, intelligent software and education website.

The visual direction should combine:

- modern editorial graphic design
- playful geometric forms
- bold typography
- modular information architecture
- software-company credibility
- education-platform accessibility
- experimental but highly structured layouts

The website must NOT feel like:

- a generic SaaS landing page
- a dark cyberpunk AI website
- a corporate consulting website
- a children's education website
- a conventional coding bootcamp template
- a Web3 or cryptocurrency website

The design should communicate:

“Technology can be professional without being intimidating.”

The overall impression should feel:

- intelligent
- experimental
- approachable
- modern
- creative
- structured
- confident


--------------------------------------------------

## 1. VISUAL LANGUAGE

Use a warm off-white canvas with bold primary colors.

Build the graphic language using simple geometric primitives:

- circles
- semicircles
- arches
- capsules
- rectangles
- quarter circles
- stars
- waves
- abstract eyes
- orbit-like shapes
- modular grids

These graphical elements should work as a reusable visual system rather than decorative illustrations.

Avoid:

- glossy 3D blobs
- random AI-generated abstract imagery
- futuristic neon gradients
- glassmorphism
- excessive shadows

Graphics should feel intentionally composed, almost like a contemporary poster or editorial magazine layout.

### 한글 줄바꿈 (Korean line-breaking)

모든 텍스트는 어절(단어) 단위로 줄바꿈한다. 어절 중간에서 글자가 잘려 다음 줄로 넘어가는 것(예: "실제 / 로", "구축합니 / 다")을 금지한다.

- 전역 규칙: `body`에 `word-break: keep-all` 적용 → 헤딩·본문·캡션·타임라인 등 모든 텍스트에 상속된다. 컴포넌트마다 개별로 다시 지정하지 않는다.
- 예외: 긴 영문 단어·URL·코드처럼 keep-all로 인해 컨테이너를 넘칠 수 있는 경우에만 해당 요소에 `break-words`(overflow-wrap: anywhere)를 국소 적용한다.
- `<br>`로 강제 줄바꿈한 헤딩도 각 줄이 어절 경계에서 끊기는지 확인한다.


### Eyebrow Label

12–14px

Uppercase or compact labeling style.

Use typography as part of the visual composition.

Headlines may:

- break across unusual lines
- align with geometric shapes
- partially overlap graphics
- occupy 40–70% of viewport width

Do not center every headline.

Prefer asymmetrical editorial placement.

## 10. MOTION SYSTEM

Motion should be playful but restrained.

Use:

- shape rotation
- slow floating movement
- horizontal marquee
- scroll-triggered shape assembly
- underline drawing
- number counting
- card reveal
- text masking transition


Animation duration:

Fast interaction:
150–250ms

UI transition:
300–500ms

Editorial section animation:
600–900ms


Avoid:

- excessive parallax
- constant bouncing
- mouse-following cursors
- aggressive scroll hijacking


--------------------------------------------------

## 11. MOBILE DESIGN

Mobile layout must not simply stack desktop components.

Recompose content intentionally.

Hero:

- typography first
- graphic shapes below headline
- CTA buttons vertically stacked if necessary

Program cards:

- one large card per row
- occasional horizontal card carousel allowed

Diagrams:

Convert horizontal process into:

- vertical timeline
or
- swipeable modular steps

Maintain generous spacing.

Do not shrink large desktop graphics until they become unreadable.


--------------------------------------------------

## FINAL DESIGN PRINCIPLE

The visual identity should sit between:

a contemporary editorial magazine,
a creative technology studio,
and a modern experimental learning platform.

The website should not say:

“We teach coding.”

It should communicate:

“We help people and organizations understand technology,
build with it,
and turn ideas into working products.”

Maintain a strong connection between:

LEARN
BUILD
EXPERIMENT
APPLY

through the entire interface.

Create a cohesive design system where typography, geometry, color, layout, interaction, and information architecture all feel like parts of one visual language.
