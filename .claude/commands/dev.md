작업 요청: $ARGUMENTS

다음 순서로 진행한다.

1. **컨텍스트 확인**: `CLAUDE.md`와 관련된 `docs/기획서.md`, `docs/디자인.md`, `docs/스펙.md` 부분을 확인한다. 이미 알고 있는 내용이면 다시 읽지 않는다.
2. **팀 구성 판단**: 요청 내용에 따라 아래 프로젝트 전용 서브에이전트 중 필요한 것을 골라 위임한다. 서로 독립적인 작업이면 병렬로 실행한다.
   - `nextjs-frontend` — 페이지/섹션/UI 구현·수정
   - `supabase-backend` — 스키마, 인증, RLS, 서버 데이터 작업
   - `design-reviewer` — UI 구현 후 디자인 시스템 대조 검증 (항상 nextjs-frontend 작업 뒤에 실행)
   - `qa-e2e` — 반응형/하이브리드 동작, 핵심 플로우(CTA, 폼, 인증) E2E 테스트
3. **최신 정보 우선**: Next.js/Supabase/Shadcn API가 불확실하면 기억에 의존하지 말고 `context7` MCP로 최신 문서를 확인한다.
4. **검증**: UI 변경이 있으면 `playwright` MCP로 mobile(375px)/tablet(768px)/desktop(1440px) 세 폭에서 실제로 렌더링을 확인한 뒤 완료로 보고한다. 데스크톱 레이아웃을 그대로 축소한 모바일 화면은 실패로 간주한다.
5. **완료 보고**: 무엇을 바꿨고, 어떤 에이전트/MCP를 썼고, 다음에 남은 일이 무엇인지 간결하게 요약한다.

작업 범위가 명확하지 않으면 곧바로 에이전트를 실행하지 말고 먼저 되묻는다.
